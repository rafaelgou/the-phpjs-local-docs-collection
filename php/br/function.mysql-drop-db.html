mysql_drop_db

(PHP 4, PHP 5)

mysql_drop_db — Exclui um banco de dados MySQL

Descrição

bool mysql_drop_db ( string $database_name [, resource $link_identifier ] )

mysql_drop_db() tenta remover todo um banco de dados do servidor associado ao identificador de conexão dado. Esta função é obsoleta, é preferível usar mysql_query() para realilar um comando sql DROP DATABASE ao invés.

Parâmetros

database_name

O nome do banco de dados que será excluído.

link_identifier

The MySQL connection. If the link identifier is not specified, the last link opened by mysql_connect() is assumed. If no such link is found, it will try to create one as if mysql_connect() was called with no arguments. If no connection is found or established, an E_WARNING level error is generated.

Valor Retornado

Retorna TRUE em caso de sucesso ou FALSE em caso de falha.

Exemplos

Exemplo #1 Exemplo alternativo de mysql_drop_db()

<?php
$link = mysql_connect('localhost', 'mysql_user', 'mysql_password');
if (!$link) {
    die('Não foi possível conectar: ' . mysql_error());
}

$sql = 'DROP DATABASE my_db';
if (mysql_query($sql, $link)) {
    echo "O banco de dados foi excluído com sucesso\n";
} else {
    echo 'Erro ao excluir o banco de dados: ' . mysql_error() . "\n";
}
?>

Notas

Aviso

Esta função não estará disponível se a extensão MySQL foi compilada com a versão da biblioteca de cliente MySQL 4.x.

Nota:

Para compatibilidade com versões anteriores, o seguinte apelido obsoleto pode ser usado: mysql_dropdb()
